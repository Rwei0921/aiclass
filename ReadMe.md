# AI Explorer：互動式 AI 入門教學網站（期中/期末成果報告規劃）

> 目標：讓「完全不懂 AI」的使用者，也能透過 **點擊互動 + 短影片 + 可視化圖形** 逐步理解 AI 概念。  
> 呈現方式：以網站作為成果報告展示（期中 50% / 期末 50%）。

---

## 1. 專案定位與核心理念

### 1.1 專案名稱
- AI Explorer（可改成：AI 新手村 / AI 互動導覽 / AI Guide）

### 1.2 目標受眾
- 完全不懂 AI 的一般使用者（同學、家人、非資訊背景）

### 1.3 核心體驗（最重要）
使用者 **點哪裡想了解** → 網頁就用「短影片 + 可視化圖 + 白話解釋」呈現  
**點 → 看 → 懂**（降低閱讀門檻）

---

## 2. 網站整體架構（Site Map）

建議至少 4 大頁面（期中可完成基本版，期末再加深互動與內容）

1. **Home 首頁**
2. **Learn 互動學習（主頁：可點擊可視化）**
3. **Cases 應用案例（生活化：醫療/交通/娛樂…）**
4. **Report 成果報告（期中/期末進度、技術架構、反思）**
（可選）5. **Glossary 名詞小字典**（讓新手隨時查）

---

## 3. 主互動頁設計：可點擊可視化 + 解釋面板

### 3.1 版面配置（推薦）
- 左側：**可點擊可視化圖**（流程圖 / 圈圈圖 / 卡片）
- 右側：**解釋面板**（影片 + 圖解 + 文字 + 小測驗）

### 3.2 可視化主題（建議主題一個就夠，期末可加第二個）
**A. AI Pipeline 流程圖（最推薦）**
- 資料蒐集 → 前處理 → 模型 → 訓練 → 評估 → 部署
- 每個節點都能點擊切換內容

（可選加強）
**B. AI / ML / DL 分層圈圈圖**
- AI 最大圈，ML 中圈，DL 內圈
- 點圈圈看差異與例子

**C. 分類 / 回歸 / 生成 卡片**
- 三張卡片，點進去看：定義、例子、常見誤解

---

## 4. 每個節點的「解釋面板」內容模板（統一格式）

點擊任一概念節點後，右側面板固定顯示：

1. **一句話白話版（大標）**
2. **30–60 秒短影片（YouTube / mp4）**
3. **可視化圖形（SVG/圖片/GIF/簡易動畫）**
4. **生活例子（1–2 個）**
5. **常見誤解（1 句澄清）**
6. **小測驗（1 題）**
   - 使用者點選 A/B/C → 立即回饋「為什麼」

> 好處：新手每次看到的版型都一樣，學習負擔低；老師看也覺得設計完整。

## 6. 內容規劃（AI Pipeline 節點清單）

> 以下為 Learn 主互動頁的 6 節點（建議期中先做基本，期末補滿模板）

1. **資料（Data）**
2. **前處理（Preprocessing）**
3. **模型（Model）**
4. **訓練（Training）**
5. **評估（Evaluation）**
6. **部署（Deployment）**

---

## 7. 影片與可視化素材策略（省時但專業）

### 7.1 影片（建議）
- 期中：先用 YouTube 嵌入（30–90 秒短片）
- 期末：可補 1–2 個自製短片（Canva/PPT 錄 1 分鐘旁白）

### 7.2 可視化（由簡到難，建議分階段）
- 期中：
  - SVG/圖片流程圖 + 點擊節點高亮
  - 簡單曲線示意圖（loss/accuracy）
- 期末：
  - 加入 GIF 動畫（例如 overfitting 示意）
  - 互動滑桿（例如資料量變多 → 表現更好）

---

## 8. 技術選型（建議路線）

### 8.1 路線 A：純前端（最穩、最適合期中）
- HTML / CSS / JavaScript
- UI：Bootstrap 或 Tailwind（擇一）
- 部署：GitHub Pages

### 8.2 路線 B：前端 + 輕後端（期末加分）
- 前端：同上
- 後端：Flask 或 FastAPI
- 可加功能：文字分類、簡單問答、互動小工具
- 部署：Render / Railway（擇一）

> 建議策略：期中走路線 A；期末再決定是否加入路線 B。
